## `exec` 回傳陣列的額外屬性完整解說

### 基本範例設定

```javascript
// 執行這段程式碼來產生 result
const result = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/.exec("2026-04-13")
```

### 額外屬性詳細表格

| 屬性名稱 | 資料型態 | 內容說明 | 範例值 | 如何存取 | 何時有值 |
|----------|----------|----------|--------|----------|----------|
| `result.index` | 數字 (Number) | 完整匹配的第一個字元在原始字串中的位置（從 0 開始計數） | `0` | `result.index` | 一定有值（匹配失敗時不會進到這裡） |
| `result.input` | 字串 (String) | 被搜尋的原始完整字串（也就是傳入 `exec()` 的那個字串） | `"2026-04-13"` | `result.input` | 一定有值（匹配失敗時不會進到這裡） |
| `result.groups` | 物件 (Object) 或 `undefined` | 儲存**命名捕獲組**的物件。若正規表達式中有使用 `(?<name>...)` 語法，這裡會將名稱對應到匹配的值 | `{ year: "2026", month: "04", day: "13" }` | `result.groups.year` | **只有**使用命名捕獲組時才有值，否則為 `undefined` |

## 詳細圖解：`result` 物件的完整結構

```javascript
// 執行程式碼
const result = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/.exec("2026-04-13")

// result 的完整內容如下：
result = [
  "2026-04-13",    // 索引 0：完整匹配
  "2026",          // 索引 1：第1個捕獲組（依括號順序）
  "04",            // 索引 2：第2個捕獲組
  "13",            // 索引 3：第3個捕獲組
  index: 0,        // 額外屬性：匹配起始位置
  input: "2026-04-13",  // 額外屬性：原始輸入字串
  groups: {        // 額外屬性：命名捕獲組物件
    year: "2026",
    month: "04",
    day: "13"
  }
]

// 如何取出各個值：
result[0]              // "2026-04-13" → 完整匹配
result[1]              // "2026" → 第1個捕獲組（年）
result[2]              // "04"   → 第2個捕獲組（月）
result[3]              // "13"   → 第3個捕獲組（日）
result.index           // 0      → 匹配從索引 0 開始
result.input           // "2026-04-13" → 原始字串
result.groups.year     // "2026" → 命名捕獲組「年」
result.groups.month    // "04"   → 命名捕獲組「月」
result.groups.day      // "13"   → 命名捕獲組「日」
```

## 每個屬性的實際應用場景

| 屬性 | 實際應用範例 | 程式碼 | 輸出結果 |
|------|--------------|--------|----------|
| `index` | 找出匹配內容在原文中的位置，用於標記或取代 | `"今天是2026-04-13好日子".match(/\d{4}-\d{2}-\d{2}/)` 的 `index` | `3`（日期從第3個字元開始） |
| `input` | 在複雜處理中記住原本的字串，避免另外儲存變數 | `result.input` 可直接取得 `"今天是2026-04-13好日子"` | 不需要額外變數 |
| `groups` | 用名稱取出值，比用 `result[1]`、`result[2]` 更直覺好讀 | `result.groups.year` 而非 `result[1]` | 程式碼更易閱讀和維護 |

## 比較：一般捕獲組 vs 命名捕獲組

| 類型 | 語法 | 存取方式 | 優點 | 範例 |
|------|------|----------|------|------|
| 一般捕獲組 | `( )` | `result[1]`、`result[2]`... | 寫法簡短 | `result[1]` → `"2026"` |
| 命名捕獲組 | `(?<name> )` | `result.groups.name` | 有名字好記，順序改變也不影響 | `result.groups.year` → `"2026"` |

```javascript
// 一般捕獲組：必須記住順序
const r1 = /(\d{4})-(\d{2})-(\d{2})/.exec("2026-04-13")
r1[1]  // "2026" → 年（第1個括號）
r1[2]  // "04"   → 月（第2個括號）
r1[3]  // "13"   → 日（第3個括號）

// 命名捕獲組：用名稱存取，順序隨便
const r2 = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/.exec("2026-04-13")
r2.groups.year   // "2026" → 直接叫名字
r2.groups.month  // "04"
r2.groups.day    // "13"
```

## 注意事項

| 注意事項 | 說明 |
|----------|------|
| `index` 和 `input` 永遠成對出現 | 只要 `exec` 成功匹配，這兩個屬性就一定存在 |
| `groups` 可能為 `undefined` | 如果正規表達式中沒有任何 `(?<name>...)`，則 `groups` 為 `undefined` |
| `match` (無 `g`) 也有這些屬性 | `"2026-04-13".match(/(\d{4})-(\d{2})-(\d{2})/)` 回傳的陣列同樣有 `index`、`input`、`groups` |
| `match` (有 `g`) 則沒有 | 加上 `g` 旗標後，只回傳簡單字串陣列，**沒有**這些額外屬性 |
