## 邊界斷言（Assertions）完整總表

邊界斷言不會實際消費（匹配）字元，而是檢查位置是否符合條件，因此也稱為「零寬斷言」。

| 斷言符號 | 名稱 | 說明 | 範例 | 匹配結果 | 不匹配結果 | 字元解說 |
|:---:|:---|:---|:---|:---|:---|:---|
| `^` | 開頭錨點 | 匹配字串開頭（多行模式下匹配行開頭） | `/^Hello/` 測試 `"Hello world"` | `"Hello"` | `" world Hello"` | `^` 匹配開頭位置，`Hello` 匹配字面字串 |
| `$` | 結尾錨點 | 匹配字串結尾（多行模式下匹配行結尾） | `/world$/` 測試 `"Hello world"` | `"world"` | `"world Hello"` | `world` 匹配字面字串，`$` 匹配結尾位置 |
| `\b` | 單字邊界 | 匹配單字與非單字之間的位置（字母/數字/底線 vs 其他） | `/\bcat\b/` 測試 `"a cat sits"` | `"cat"` | `"scat"` 或 `"cats"` | `\b` 匹配單字邊界，`cat` 匹配字面字串，`\b` 匹配單字邊界 |
| `\B` | 非單字邊界 | 匹配非單字邊界的位置（單字內部） | `/\Bcat\B/` 測試 `"scatty"` | `"cat"` | `"a cat"` | `\B` 匹配非單字邊界，`cat` 匹配字面字串，`\B` 匹配非單字邊界 |
| `(?=p)` | 正向先行斷言 | 匹配後面緊跟著 `p` 的位置（但不消費 `p`） | `/\d+(?=px)/` 測試 `"100px"` | `"100"` | `"100pt"` | `\d+` 匹配數字，`(?=px)` 要求後面必須是 `px` |
| `(?!p)` | 負向先行斷言 | 匹配後面不緊跟著 `p` 的位置 | `/\d+(?!px)/` 測試 `"100pt"` | `"100"` | `"100px"` | `\d+` 匹配數字，`(?!px)` 要求後面不能是 `px` |
| `(?<=p)` | 正向後行斷言 | 匹配前面緊跟著 `p` 的位置（ES2018） | `/(?<=\$)\d+/` 測試 `"$100"` | `"100"` | `"100"` | `(?<=\$)` 要求前面必須是 `$`，`\d+` 匹配數字 |
| `(?<!p)` | 負向後行斷言 | 匹配前面不緊跟著 `p` 的位置（ES2018） | `/(?<!\$)\d+/` 測試 `"100"` | `"100"` | `"$100"` | `(?<!\$)` 要求前面不能是 `$`，`\d+` 匹配數字 |

---

### 邊界斷言詳細對照表（含多行模式說明）

| 斷言符號 | 正常模式 | 多行模式（`m` 旗標） |
|:---:|:---|:---|
| `^` | 僅匹配整個字串開頭 | 匹配每行開頭（換行符後的位置） |
| `$` | 僅匹配整個字串結尾 | 匹配每行結尾（換行符前的位置） |


---

### 先行斷言 vs 後行斷言對照表

| 比較項目 | 先行斷言 `(?=p)` / `(?!p)` | 後行斷言 `(?<=p)` / `(?<!p)` |
|:---|:---|:---|
| 檢查方向 | 往**右**檢查（後面） | 往**左**檢查（前面） |
| ES 版本 | ES3（早期就支援） | ES2018（較新，舊環境不支援） |
| 常見應用 | 擷取特定單位前的數值（如 `100px` → `100`） | 擷取特定貨幣後的數值（如 `$100` → `100`） |
| 範例 | `/\d+(?=px)/` | `/(?<=\$)\d+/` |

---

### 邊界斷言實際應用範例表

| 應用場景 | 正規表達式 | 測試字串 | 匹配結果 | 字元解說 |
|:---|:---|:---|:---|:---|
| 擷取開頭的第一個單字 | `/^\w+/` | `"Hello world"` | `"Hello"` | `^` 字串開頭，`\w+` 一個以上單字字元 |
| 擷取結尾的最後一個單字 | `/\w+$/` | `"Hello world"` | `"world"` | `\w+` 一個以上單字字元，`$` 字串結尾 |
| 找出獨立單字 "is" | `/\bis\b/` | `"this is a test"` | `"is"` | `\b` 單字邊界，`is` 字面字串，`\b` 單字邊界 |
| 擷取 CSS 中的像素值 | `/\d+(?=px)/` | `"width: 100px"` | `"100"` | `\d+` 數字，`(?=px)` 後面必須是 `px` |
| 擷取非像素值的數字 | `/\d+(?!px)/` | `"100px 50%"` | `"50"` | `\d+` 數字，`(?!px)` 後面不能是 `px` |
| 擷取金額數字 | `/(?<=\$)\d+/` | `"price: $100"` | `"100"` | `(?<=\$)` 前面必須是 `$`，`\d+` 數字 |
| 擷取非金額數字 | `/(?<!\$)\d+/` | `"$100 50"` | `"50"` | `(?<!\$)` 前面不能是 `$`，`\d+` 數字 |
| 驗證密碼至少一個數字 | `/^(?=.*\d)/` | `"abc123"` | 空（只檢查位置） | `^` 開頭，`(?=.*\d)` 後面至少有一個數字 |

---

### 注意事項表

| 注意事項 | 說明 |
|:---|:---|
| 零寬特性 | 斷言不消費字元，匹配後游標位置不移動 |
| 後行斷言相容性 | `(?<=p)` 和 `(?<!p)` 需要 ES2018+，舊瀏覽器不支援 |
| 巢狀斷言 | 斷言內部可以包含其他斷言，如 `/(?=(?=a))/` |
| 效能考量 | 複雜的斷言（特別是後行斷言）可能影響效能 |

## 五、Set

> **用途**：去重、集合運算（聯集、交集、差集）。

```js
const set = new Set([1, 2, 2, 3, 3]); // {1, 2, 3} ← 自動去重

// 基本操作
set.add(4);        // 加入值
set.has(2);        // true
set.delete(2);     // 刪除，回傳 boolean
set.size;          // 項目數量
set.clear();       // 清空

// 遍歷
for (const val of set) { console.log(val); }
[...set];          // 轉成陣列

// 集合運算
const A = new Set([1, 2, 3, 4]);
const B = new Set([3, 4, 5, 6]);

// 聯集 (Union)
const union = new Set([...A, ...B]);        // {1,2,3,4,5,6}

// 交集 (Intersection)
const intersection = new Set([...A].filter(x => B.has(x))); // {3,4}

// 差集 (Difference) A - B
const difference = new Set([...A].filter(x => !B.has(x)));  // {1,2}
