這是一份針對我們討論至今的所有 JavaScript 正規表達式（Regex）教學內容的系統化整理。我將其歸類為 **「基礎語法」**、**「核心方法」**、**「修飾符」** 以及 **「進階控制」** 四大模組：

---

## 🛠 一、 基礎語法模組 (Patterns)
定義「要找什麼」的核心規則。

| 類別 | 符號 | 邏輯意義 | 範例 |
| :--- | :--- | :--- | :--- |
| **基本字元** | `.` | 匹配除換行外的**任一字元**。 | `.T.` 匹配 `ATA`, `CTC` |
| **字元類別** | `\d`, `\w`, `\s` | 匹配**數字**、**單字字元**（含底線）、**空白**。 | `\d+` 匹配 `100` |
| **集合與範圍** | `[]`, `[^]` | **集合**（匹配內含任一字元）或**否定**（不含）。 | `[Ee]` 匹配大小寫 E |
| **量詞** | `*`, `+`, `{n,m}` | **0次以上**、**1次以上**、**指定次數區間**。 | `\w{3,5}` 匹配 3-5 字 |
| **位置錨點** | `^`, `$` | 限制必須在**行首**或**行尾**。 | `^Loid` 必須開頭是 Loid |
| **跳脫符號** | `\` | 將特殊功能符號**轉義為純文字**。 | `\.` 匹配點號本身 |

---

## ⚙️ 二、 核心方法模組 (Methods)
定義「如何執行」比對動作。

| 方法 | 所屬對象 | 嚴謹定義 | 常用場景 |
| :--- | :--- | :--- | :--- |
| **`test()`** | `RegExp` | **判定性檢索**：回傳 `Boolean`。 | 欄位格式驗證（如 Email） |
| **`exec()`** | `RegExp` | **詳細性檢索**：回傳 `Array` 與位置、分組。 | 需要提取括號內詳細內容 |
| **`match()`** | `String` | **集合性檢索**：受 `g` 標記影響回傳深度。 | 一次抓出字串內所有符合項 |
| **`search()`** | `String` | **定位性檢索**：回傳第一個符合項的 `Index`。 | 尋找關鍵字在字串的位置 |
| **`replace()`** | `String` | **置換應用**：配合捕獲組重新排列字串。 | 格式轉換（如姓氏名字對調） |
| **`matchAll()`** | `String` | **全域詳細檢索**：回傳所有匹配項的迭代器。 | 多行且需要每組捕獲細節時 |

---

## 🚩 三、 修飾符模組 (Flags)
定義「搜尋環境」的設定。

* **`i` (Ignore Case)**：忽略大小寫差異。
* **`g` (Global)**：全域搜尋，不因找到第一個就停止。
* **`m` (Multiline)**：多行模式，讓 `^` 與 `$` 作用於每一行的開頭結尾。
* **`s` (dotAll)**：讓點號 `.` 可以匹配換行符。
* **`u` (Unicode)**：正確處理 Emoji 或特殊編碼字元。
* **`y` (Sticky)**：粘性匹配，強制從 `lastIndex` 索引位置精確起始。

---

## 🚀 四、 進階控制模組 (Grouping & Logic)
定義「如何操控」匹配到的數據。

### 1. 捕獲組 (Capturing Groups) `( )`
* **功能**：資訊提取、反向引用。
* **特性**：會佔用記憶體儲存結果，可透過 `result[1]` 或 `$1` 存取。

### 2. 非捕獲組 (Non-capturing Groups) `(?: )`
* **功能**：純邏輯分組。
* **特性**：**不儲存結果**、**節省記憶體**、**優化效能**。

### 3. 反向引用 (Back-reference) `\1`
* **功能**：在 Regex 內部重複使用前面抓到的內容。
* **範例**：標籤成對檢查 `<a>...</a>`。

---

### 💡 學習總結
正規表達式的精髓在於：
1.  用 **Patterns** 描述形狀。
2.  用 **Flags** 設定環境。
3.  用 **Methods** 啟動搜尋。
4.  用 **Groups** 提取並利用數據。

這份清單涵蓋了從初階到中高階開發者日常所需的所有 Regex 核心知識點。
