
# 🏆 正規表達式 (Regex) 終極圖解與實戰指南

## 第一部分：核心特殊符號總表（優化補全版）

> **💡 閱讀提示**：符號包在 `/ /` 中代表正規表達式的開始與結束。

### 1. 量詞（出現的次數）
控制前面的字元可以出現幾次。

| 符號 | 意義 | 💡 實戰範例 | 🔍 拆解與比對結果 |
|:---:|:---|:---|:---|
| **`*`** | **0 次或多次**<br>(⚠️*之前漏掉的超級核心*) | `/go*d/` | 允許 `o` 沒有出現，或出現無限次。<br>✅ 匹配：`gd`, `god`, `good`, `goood`<br>❌ 失敗：`gxd` |
| **`+`** | **1 次或多次** | `/go+d/` | `o` 至少要出現 1 次。<br>✅ 匹配：`god`, `good`<br>❌ 失敗：`gd` (因為 o 沒出現) |
| **`?`** | **0 次或 1 次**<br>(可有可無) | `/colou?r/` | `u` 只能出現 0 次或 1 次。<br>✅ 匹配：`color`, `colour`<br>❌ 失敗：`colouur` |
| **`{n}`** | **剛好 n 次** | `/a{3}/` | `a` 必須精準連續出現 3 次。<br>✅ 匹配：`aaa`<br>❌ 失敗：`aa` |
| **`{n,}`** | **至少 n 次** | `/a{2,}/` | `a` 至少 2 次以上（無上限）。<br>✅ 匹配：`aa`, `aaa`, `aaaa` |
| **`{n,m}`** | **出現 n 到 m 次** | `/a{1,3}/` | `a` 出現 1 到 3 次。<br>✅ 匹配：`a`, `aa`, `aaa`<br>❌ 失敗：`aaaa` (超出次數) |

### 2. 邊界與錨點（定位點）
不匹配具體字元，而是匹配「位置」。

| 符號 | 意義 | 💡 實戰範例 | 🔍 拆解與比對結果 |
|:---:|:---|:---|:---|
| **`^`** | **字串開頭** | `/^Apple/` | 必須以 Apple **開頭**。<br>✅ 匹配：`Apple is good`<br>❌ 失敗：`I like Apple` |
| **`$`** | **字串結尾** | `/Apple$/` | 必須以 Apple **結尾**。<br>✅ 匹配：`I like Apple`<br>❌ 失敗：`Apple is good` |
| **`\b`** | **單字邊界** | `/\byou\b/` | 尋找獨立的單字 "you"。<br>✅ 匹配：`I love you so much`<br>❌ 失敗：`your book` (因為 you 連著 r) |
| **`\B`** | **非單字邊界** | `/you\B/` | you 後面必須連著其他字母。<br>✅ 匹配：`your`, `youth`<br>❌ 失敗：`you` |

### 3. 字元類別與集合（找特定種類的字）
用簡短的符號代表一整群字元。大寫通常是小寫的「相反」。

| 符號 | 意義 | 等同於 | 說明與範例 |
|:---:|:---|:---:|:---|
| **`\d`** | **數字** (digit) | `[0-9]` | 只要是數字就抓。如 `/\d+/` 可抓到 `123` |
| **`\D`** | **非數字** | `[^0-9]` | 抓取英文、中文、符號等**不是數字**的東西 |
| **`\w`** | **單字字元** (word) | `[A-Za-z0-9_]` | 包含英文字母、數字與**底線**。常用于帳號驗證 |
| **`\W`** | **非單字字元** | `[^A-Za-z0-9_]` | 抓取標點符號、空白、特殊符號（如 `@`, `#`） |
| **`\s`** | **空白字元** (space) | `[ \t\n\r\f]` | 抓取所有的空格、Tab、換行符號 |
| **`\S`** | **非空白字元** | `[^ \t\n\r\f]` | 只要不是空白的任何字元都抓 |
| **`.`** | **任意字元** | - | 萬用字元！可代表除換行符 `\n` 外的**任何一個字元** |
| **`[ ]`** | **自訂集合** | - | `/[aeiou]/`：只要是 a,e,i,o,u 其中**一個**就符合 |
| **`[^ ]`** | **排除集合** | - | `/[^aeiou]/`：只要**不是** a,e,i,o,u 的字元就符合 |

### 4. 群組、條件與其他控制
| 符號 | 意義 | 💡 實戰範例 | 🔍 拆解與比對結果 |
|:---:|:---|:---|:---|
| **`\|`** | **或 (OR)** | `/cat\|dog/` | 滿足其中一個即可。<br>✅ 匹配：`cat` 或 `dog` |
| **`( )`** | **群組 / 捕獲** | `/(ab)+/` | 將 `ab` 視為一個整體。<br>✅ 匹配：`ab`, `abab`, `ababab` |
| **`\`** | **跳脫字元**<br>(⚠️*新增*) | `/\./` 或 `/\+/` | Regex 符號若要當作普通文字尋找，前面需加 `\`。<br>✅ `/\./` 專門尋找真正的點號 `.` |

## 第二部分：豐富度擴充（進階核心觀念）

為了讓這份筆記能真正運用在實戰中，為您補充三個必學的 Regex 觀念：

### 📌 1. 修飾符 (Flags)：改變搜尋的規則
通常寫在正規表達式的最後面，例如 `/pattern/g`。
*   **`g` (Global - 全域搜尋)**：預設情況下，Regex 找到第一個符合的字串就會停下來。加上 `g` 就能把整篇文章所有符合的都找出來。
*   **`i` (Ignore Case - 忽略大小寫)**：加上 `i` 後，`/a/i` 可以同時匹配 `A` 和 `a`。
*   **`m` (Multiline - 多行模式)**：讓 `^` 和 `$` 不只匹配整份文件的開頭結尾，而是能匹配**每一行**的開頭和結尾。

### 📌 2. 貪婪 (Greedy) vs 懶惰 (Lazy) 模式
Regex 預設是**「貪婪的」**，它會盡可能抓取最長的字串。若要它變**「懶惰」**（抓最短就停），可以在量詞後面加上 `?`。

*   **情境**：從字串 `<div>hello</div><div>world</div>` 中抓取 div 標籤。
*   👿 **貪婪模式 `/<.*>/`**：它會一路從第一個 `<` 抓到整行最後一個 `>`。
    *   結果：抓到 `<div>hello</div><div>world</div>` (整個字串)。
*   😇 **懶惰模式 `/<.*?>/`**：加上 `?`，告訴 Regex「遇到第一個符合的就停下來」。
    *   結果：只抓到 `<div>`。

## 第三部分：常見實戰套用範例 (Copy & Paste 備用區)

有了上面的積木（符號），我們就能組合成實用的檢查工具：

1. **僅限輸入數字**
   `/^\d+$/`
   *(解說：開頭 `^` 到結尾 `$` 中間，必須由 1 個以上的數字 `\d+` 組成)*

2. **驗證台灣手機號碼 (09開頭，共 10 碼)**
   `/^09\d{8}$/`
   *(解說：開頭必須是 09，後面緊接剛好 8 個數字 `\d{8}`，然後結尾)*

3. **包含大小寫英文字母與數字 (長度 6-12)**
   `/^[A-Za-z0-9]{6,12}$/`
   *(解說：使用集合 `[ ]` 限制字元種類，並用 `{6,12}` 限制長度)*

| `\S` | 非空白字元 | `[^ \t\n\r\f]` | 只要不是空白的任何字元都抓 | `/* 範例：/\S+/ 執行於 "hello world" */`<br>`\S : 匹配非空白字元 h`<br>`+ : 量詞，繼續匹配 e、l、l、o`<br>✅ 匹配結果：`"hello"`（遇到空格停止）<br>❌ 不匹配：空格本身 |

| `[ ]` | 自訂集合 | - | `/[aeiou]/`：只要是 a,e,i,o,u 其中一個就符合 | `/* 範例：/[aeiou]/ 執行於 "hello" */`<br>`[aeiou] : 字元集合，匹配 a、e、i、o、u 其中之一`<br>✅ 匹配結果：`"e"`（h 不符合，e 符合）<br>❌ 不匹配：`"h"`、`"l"`、`"l"`、`"o"`（o 雖是母音但 e 先匹配） |
| `[^ ]` | 排除集合 | - | `/[^aeiou]/`：只要不是 a,e,i,o,u 的字元就符合 | `/* 範例：/[^aeiou]/ 執行於 "hello" */`<br>`[^aeiou] : 排除集合，匹配非母音字母`<br>✅ 匹配結果：`"h"`（h 不是母音）<br>❌ 不匹配：`"e"`、`"o"`（母音被排除） |

| 符號 | 意義 | 💡 實戰範例 | 🔍 拆解與比對結果 |
|:---:|:---|:---|:---|

### 三、邊界錨點（補充）

| 符號 | 意義 | 💡 實戰範例 | 🔍 拆解與比對結果 |
|:---:|:---|:---|:---|
| `^` | 字串開頭 | `/^Apple/` | `/* 執行於 "Apple is good" */`<br>`^ : 錨點，必須從字串開頭開始比對`<br>`A : 匹配字母 "A"`<br>`p : 匹配 "p"`<br>`p : 匹配 "p"`<br>`l : 匹配 "l"`<br>`e : 匹配 "e"`<br>✅ 匹配結果：`"Apple"`<br><br>`/* 執行於 "I like Apple" */`<br>❌ 失敗：字串開頭是 "I"，不是 "Apple" |
| `$` | 字串結尾 | `/Apple$/` | `/* 執行於 "I like Apple" */`<br>`A : 匹配 "A"`<br>`p : 匹配 "p"`<br>`p : 匹配 "p"`<br>`l : 匹配 "l"`<br>`e : 匹配 "e"`<br>`$ : 錨點，必須在字串結尾`<br>✅ 匹配結果：`"Apple"`<br><br>`/* 執行於 "Apple is good" */`<br>❌ 失敗：結尾是 "good"，不是 "Apple" |
| `\b` | 單字邊界 | `/\bcat\b/` | `/* 執行於 "The cat is here" */`<br>`\b : 單字邊界（單字開頭）`<br>`c : 匹配 "c"`<br>`a : 匹配 "a"`<br>`t : 匹配 "t"`<br>`\b : 單字邊界（單字結尾）`<br>✅ 匹配結果：`"cat"`<br><br>`/* 執行於 "The catcat is here" */`<br>❌ 失敗：catcat 中間沒有邊界 |
| `\B` | 非單字邊界 | `/\Bcat\B/` | `/* 執行於 "The catcat is here" */`<br>`\B : 非單字邊界（單字內部）`<br>`c : 匹配 "c"`<br>`a : 匹配 "a"`<br>`t : 匹配 "t"`<br>`\B : 非單字邊界`<br>✅ 匹配結果：`"cat"`（第二個 cat 位於單字內部）<br><br>`/* 執行於 "The cat is here" */`<br>❌ 失敗：cat 前後是邊界，不符合 \B |
