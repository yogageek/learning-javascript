## 捕獲組（Capturing Group）範例表

| 概念 | 說明 | 範例 | 字元解說 |
|------|------|------|----------|
| **捕獲組基本用法** | 用小括號 `( )` 將部分正規表達式包起來，可單獨提取該部分匹配的內容 | `/(\d+)/.exec("abc123")` → `["123", "123"]` | `/*`<br>`( : 捕獲組起始`<br>`\d : 匹配任何數字字元 (0-9)`<br>`+ : 量詞，匹配前一個字元 1 次或多次`<br>`) : 捕獲組結束`<br>`完整匹配為 "123"，捕獲組也是 "123"（本例兩者相同）`<br>`*/` |
| **多個捕獲組** | 同時提取多個不同區塊的內容 | `/(\d{4})-(\d{2})-(\d{2})/.exec("2026-04-13")` → `["2026-04-13", "2026", "04", "13"]` | `/*`<br>`(\d{4}) : 第1個捕獲組，匹配年份`<br>`- : 匹配連字號`<br>`(\d{2}) : 第2個捕獲組，匹配月份`<br>`- : 匹配連字號`<br>`(\d{2}) : 第3個捕獲組，匹配日期`<br>`回傳陣列：[完整匹配, 年, 月, 日]`<br>`*/` |
| **實際應用：提取姓名和電話** | 從字串中擷取不同類型的資訊 | `/(\w+): (\d{10})/.exec("小明: 0912345678")` → `["小明: 0912345678", "小明", "0912345678"]` | `/*`<br>`(\w+) : 第1個捕獲組，匹配姓名（字母/數字/底線）`<br>`: : 匹配冒號和空格`<br>`(\d{10}) : 第2個捕獲組，匹配10位數字電話號碼`<br>`*/` |
| **實際應用：提取 Email 帳號和域名** | 從 Email 中分別取出@前後的內容 | `/(.+)@(.+)/.exec("user@example.com")` → `["user@example.com", "user", "example.com"]` | `/*`<br>`(.+) : 第1個捕獲組，匹配 @ 前的帳號名稱`<br>`@ : 匹配 @ 符號`<br>`(.+) : 第2個捕獲組，匹配 @ 後的域名`<br>`*/` |

## 捕獲組只能搭配 `exec` 嗎？

**不是的。** 捕獲組可以搭配多種方法使用，只要該方法會回傳捕獲組的詳細資訊：

| 方法 | 是否支援捕獲組 | 說明 |
|------|---------------|------|
| `exec` | ✅ 是 | 回傳陣列包含完整匹配 + 所有捕獲組 |
| `match` (無 `g` 旗標) | ✅ 是 | 行為與 `exec` 類似，回傳完整匹配 + 捕獲組 |
| `match` (有 `g` 旗標) | ❌ 否 | 只回傳完整匹配的字串陣列，**不包含**捕獲組 |
| `test` | ❌ 否 | 只回傳 `true/false`，不回傳捕獲組內容 |
| `search` | ❌ 否 | 只回傳索引位置，不回傳捕獲組內容 |
| `replace` | ✅ 是 | 可在替換時用 `$1`, `$2` 等參照捕獲組 |
| `matchAll` | ✅ 是 | 回傳迭代器，每個元素都包含捕獲組 |


## 其他方法支援捕獲組對照表

| 方法 | 是否支援捕獲組 | 說明 | 完整範例與註解 |
|------|---------------|------|----------------|
| `exec` | ✅ 是 | 回傳陣列包含完整匹配 + 所有捕獲組 | `/(\d{4})-(\d{2})-(\d{2})/.exec("2026-04-13")` → `["2026-04-13", "2026", "04", "13"]`<br>/*<br>  exec 回傳陣列：<br>    [0] → 完整匹配 "2026-04-13"<br>    [1] → 第1個捕獲組 "2026"<br>    [2] → 第2個捕獲組 "04"<br>    [3] → 第3個捕獲組 "13"<br>  捕獲組 ✅ 完整保留<br>*/ |
| `match`<br>(無 `g`) | ✅ 是 | 行為與 `exec` 類似，回傳完整匹配 + 捕獲組 | `"2026-04-13".match(/(\d{4})-(\d{2})-(\d{2})/)` → `["2026-04-13", "2026", "04", "13"]`<br>/*<br>  match 無 g 時，行為與 exec 完全相同：<br>    [0] → 完整匹配 "2026-04-13"<br>    [1] → 第1個捕獲組 "2026"<br>    [2] → 第2個捕獲組 "04"<br>    [3] → 第3個捕獲組 "13"<br>  捕獲組 ✅ 完整保留<br>*/ |
| `match`<br>(有 `g`) | ❌ 否 | 只回傳完整匹配的字串陣列，**不包含**捕獲組 | `"2026-04-13".match(/(\d{4})-(\d{2})-(\d{2})/g)` → `["2026-04-13"]`<br>/*<br>  match 有 g 時，只回傳完整匹配的字串：<br>    [0] → "2026-04-13"<br>  捕獲組 ❌ 全部遺失！<br>  即使正規表達式中有括號，也拿不到年、月、日的個別值<br>*/ |
| `test` | ❌ 否 | 只回傳 `true/false`，不回傳捕獲組內容 | `/(\d+)/.test("abc123")` → `true`<br>/*<br>  test 只回答「有沒有符合？」<br>  雖然有捕獲組 (\d+)，但無法取得 "123" 這個值<br>  捕獲組 ❌ 無法取得<br>*/ |
| `search` | ❌ 否 | 只回傳索引位置，不回傳捕獲組內容 | `"abc123".search(/(\d+)/)` → `3`<br>/*<br>  search 只回答「符合的位置在哪？」<br>  回傳數字 3（數字 "123" 從索引 3 開始）<br>  雖然有捕獲組，但無法取得 "123" 這個值<br>  捕獲組 ❌ 無法取得<br>*/ |
| `replace` | ✅ 是 | 可在替換時用 `$1`, `$2` 等參照捕獲組 | `"2026-04-13".replace(/(\d{4})-(\d{2})-(\d{2})/, "$2/$3/$1")` → `"04/13/2026"`<br>/*<br>  replace 支援在第二個參數中使用捕獲組：<br>    $1 → 第1個捕獲組 "2026"<br>    $2 → 第2個捕獲組 "04"<br>    $3 → 第3個捕獲組 "13"<br>  將日期格式從 YYYY-MM-DD 轉換為 MM/DD/YYYY<br>  捕獲組 ✅ 可用 $1,$2,$3 參照<br>*/ |
| `matchAll` | ✅ 是 | 回傳迭代器，每個元素都包含完整匹配 + 捕獲組 | `[...("2026-04-13".matchAll(/(\d{4})-(\d{2})-(\d{2})/g))]` → `[["2026-04-13", "2026", "04", "13"]]`<br>/*<br>  matchAll 即使有 g 旗標，仍然保留捕獲組：<br>    回傳迭代器，展開後得到陣列<br>    每個元素格式與 exec 相同：<br>      [0] → 完整匹配 "2026-04-13"<br>      [1] → 第1個捕獲組 "2026"<br>      [2] → 第2個捕獲組 "04"<br>      [3] → 第3個捕獲組 "13"<br>  捕獲組 ✅ 完整保留（克服了 match 有 g 的缺點）<br>*/ |

## 重點總結對照表

| 情境 | 使用的方法 | 能否取得捕獲組 | 備註 |
|------|------------|----------------|------|
| 只要知道「有沒有符合」 | `test` | ❌ 否 | 最快速 |
| 只要知道「符合的位置」 | `search` | ❌ 否 | 回傳索引 |
| 只要知道「符合的字串」（不須細節） | `match` + `g` | ❌ 否 | 簡單明快 |
| 需要「完整匹配 + 捕獲組細節」（第一次匹配） | `exec` 或 `match` 無 `g` | ✅ 是 | 兩者行為相同 |
| 需要「完整匹配 + 捕獲組細節」（多次匹配） | `matchAll` | ✅ 是 | 比迴圈執行 `exec` 更方便 |
| 需要「替換字串並參照捕獲組」 | `replace` | ✅ 是 | 用 `$1`, `$2` 等變數 |



### 額外屬性（僅 `exec` 回傳的陣列包含）

| 屬性名稱 | 內容說明 | 範例值 |
|----------|----------|--------|
| `result.index` | 完整匹配在原始字串中的起始索引位置 | `0`（若 "2026-04-13" 從頭開始） |
| `result.input` | 原始輸入字串（被搜尋的完整字串） | `"2026-04-13"` |
| `result.groups` | 命名捕獲組物件（若有使用 `(?<name>...)` 語法） | `{ year: "2026", month: "04", day: "13" }` |
